# Cara Menggunakan Tanda Tangan Pesan Omnichannel

Tanda tangan pesan pribadi berfungsi sebagai penutup komprehensif yang bisa Anda sertakan di akhir atau footer pesan Anda. Di Katalis.app, Anda bisa menggunakan tanda tangan pesan di semua inbox dan tidak terbatas hanya pada email.

Untuk membuat dan menggunakan tanda tangan Anda, ikuti langkah-langkah sederhana di bawah ini.

## Membuat Tanda Tangan Pesan yang Dipersonalisasi

Setiap agen di Katalis.app bisa membuat tanda tangan pesan sendiri. Ikuti langkah-langkah berikut.

**Langkah 1.** Klik ikon profil Anda dari kiri bawah dashboard, dan buka 'Personal Settings'.

**Langkah 2.** Gulir ke bawah ke bagian kedua halaman yang disebut 'Personal message signature'. Gunakan editor teks kaya di sini untuk membuat tanda tangan Anda. Manfaatkan berbagai opsi pemformatan, emoji, dan opsi unggah gambar untuk membuat tanda tangan yang bernilai.

**Langkah 3.** Klik tombol 'Save message signature' untuk mengunci pengaturan Anda.

## Menggunakan Tanda Tangan Pesan Anda

Anda bisa sepenuhnya mengontrol perilaku tanda tangan pesan dari dashboard Katalis.app. Tanda tangan Anda diaktifkan secara default. Saat berinteraksi dengan pelanggan, Anda memiliki tiga opsi berikut.

### Kirim pesan
Anda bisa mengetik pesan dan menjalankan alur kerja reguler Anda. Tanda tangan pesan akan secara otomatis terus ditambahkan ke semua pesan Anda.

### Edit tanda tangan sebelum mengirim
Jika Anda perlu mengedit tanda tangan—misalnya, menghapus nomor telepon, atau menghapus tanda tangan sama sekali—Anda bisa melakukannya menggunakan editor teks pesan. Setelah selesai, tekan tombol 'Send'.

### Nonaktifkan tanda tangan
Jika Anda tidak ingin menambahkan tanda tangan di pesan, cukup nonaktifkan menggunakan tombol tanda tangan yang sama.

**Penting: Kemampuan tanda tangan per inbox**

Tanda tangan pesan bervariasi tampilannya di berbagai inbox karena batasan yang berbeda yang diberlakukan oleh berbagai API channel. Secara khusus, gambar in-line hanya akan terlihat di inbox Email, live chat website, dan API, di mana teks dan gambar didukung. Sebaliknya, semua inbox lain hanya akan mengakomodasi komponen teks dari tanda tangan.
